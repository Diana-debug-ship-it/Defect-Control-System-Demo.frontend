# Мониторинг конвейера ОТК — Интерфейс оператора (Демонстрационная версия)

Данный микросервис представляет собой веб-интерфейс (SPA) на базе фреймворка React для демонстрационной версии (MVP) системы автоматизированного технического контроля (ОТК). Модуль визуализирует результаты компьютерного зрения, рассчитывает оперативную статистику смены и выводит хронологическую таблицу проверок рам в режиме реального времени.

Проект разработан в рамках производственной практики.

## Архитектурная роль в киберфизическом комплексе
1. **Инициализация интерфейса (Уровень базы данных)**: При загрузке страницы или её обновлении (`F5`) React отправляет синхронные HTTP-запросы (`fetch`) к центральному Java-бэкенду (`GET /stats` и `GET /history`). Это позволяет мгновенно восстановить честные показатели и историю проверок по всей базе данных.
2. **Мониторинг в реальном времени (Уровень потока данных)**: Сервис открывает постоянный односторонний канал Server-Sent Events (SSE) по адресу `/stream`. При фиксации брака нейросетью событие через Kafka долетает до Java и мгновенно «пушится» на фронтенд. Интерфейс реактивно добавляет новую строку в начало таблицы без перезагрузки страницы.
3. **Локализация системных данных**: Модуль содержит встроенный слой маппинга (`DEFECT_TRANSLATIONS`), который на лету переводит строгие английские коды из базы данных в понятные оператору цеха русскоязычные технологические термины.


Сервис контроля качества: [Репозиторий сервиса контроля качества](https://github.com/Diana-debug-ship-it/Defect-Control-System-Demo.Quality-control)

Сервис контроля дефектов: [Репозиторий сервиса контроля дефектов](https://github.com/Diana-debug-ship-it/Defect-Control-System-Demo.CV-service)

## 🛠 Технологический стек
* **React 18 / 19** (компонентный интерфейс)
* **Vite** (высокопроизводительный сборщик проекта)
* **JavaScript (ES6+)** + **React Hooks** (`useState`, `useEffect` для работы с таймерами и SSE-клиентом)
* **CSS Grid / Flexbox** (строгая промышленная верстка в стиле диспетчерских терминалов)

## 📊 Отображаемые параметры и KPI
* **Панель KPI (Верхний уровень)**:
  * Общее количество проверенных рам (синхронизировано с `COUNT` в БД).
  * Количество годных деталей (статус `NORMAL`).
  * Количество выявленного брака (статус `DEFECT`).
  * Текущий процент брака на линии с округлением до десятых.
* **Таблица истории (Нижний уровень)**:
  * Серийный номер рамы (уникальный ID).
  * Точное время регистрации на посту контроля.
  * Результат анализа геометрии (с расшифровкой типа брака: трещина, дефект шва, нарушение геометрии лонжерона, отсутствие отверстия).
  * Финальный прямоугольный промышленный бадж статуса ОТК с цветовым маркером.

## 🏁 Порядок запуска

1. Убедитесь, что установлена платформа **Node.js** версии 20+.
2. Установите необходимые зависимости из корневой папки фронтенда:
   ```bash
   npm install
   ```
3. Убедитесь, что Java-бэкенд запущен на порту `8080`, а в его контроллере активна аннотация `@CrossOrigin` для обхода политик безопасности браузера.
4. Запустите локальный сервер разработки Vite:
   ```bash
   npm run dev
   ```
5. Откройте предоставленную сборщиком ссылку в браузере (по умолчанию `http://localhost:5173/`).
